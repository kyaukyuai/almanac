# v0.8 RC Smoke

This smoke validates v0.8 answer diagnostics, replay, and readiness. Use fresh
disposable roots. Deterministic replay checks do not require provider
credentials. Real provider ask checks require `ANTHROPIC_API_KEY`.

Run from the repository root.

## 1. Static Gates

```bash
git switch main
git pull --ff-only

bun run typecheck
bun test
```

## 2. sqlite-demo Answer Diagnostics

Create a fresh offline demo:

```bash
tmp="$(mktemp -d -t almanac-v080-sqlite-smoke)"

bun src/cli.ts demo --root "$tmp"
bun src/cli.ts inspect sqlite-demo --root "$tmp"
bun src/cli.ts profile sqlite-demo --root "$tmp"
bun src/cli.ts benchmark sqlite-demo --root "$tmp"
```

Run a deterministic mock-provider answer and save the artifact:

```bash
export ALMANAC_LLM=mock
export ALMANAC_MOCK_RESPONSES='{
  "answer-planner@planner-v1": [
    "{\"action\":\"call_tool\",\"toolName\":\"query_facts\",\"input\":{\"q\":\"transactions atomic\",\"limit\":3}}",
    "{\"action\":\"stop\",\"reason\":\"enough-evidence\"}"
  ],
  "answer-synthesis@synthesis-v1": "{\"status\":\"ok\",\"answer\":\"SQLite transactions are atomic.\",\"citations\":[{\"sourceId\":\"sqlite-transactions\",\"url\":\"https://www.sqlite.org/lang_transaction.html\",\"fetchedAt\":\"2026-01-01T00:00:00.000Z\"}]}"
}'

bun src/cli.ts ask sqlite-demo \
  "Are SQLite transactions atomic?" \
  --save \
  --label rc-answer \
  --json \
  --root "$tmp"
```

Inspect the saved answer trace:

```bash
answer_id="$(
  bun src/cli.ts runs sqlite-demo --kind answer --latest --json --root "$tmp" \
    | jq -r '.runs[0].runId'
)"

bun src/cli.ts runs sqlite-demo "$answer_id" --json --root "$tmp"
bun src/cli.ts runs sqlite-demo "$answer_id" --root "$tmp"
```

Replay the saved answer without a provider:

```bash
unset ALMANAC_LLM
unset ALMANAC_MOCK_RESPONSES

bun src/cli.ts ask-replay sqlite-demo \
  --from-runs \
  --label rc-answer \
  --json \
  --root "$tmp"
```

Add a fixture replay row and verify answer readiness:

```bash
mkdir -p "$tmp/sqlite-demo/tests"
cat > "$tmp/sqlite-demo/tests/ask.jsonl" <<'JSONL'
{"id":"sqlite-transactions-ok","question":"Are SQLite transactions atomic?","toolCalls":[{"tool":"query_facts","input":{"q":"transactions atomic"},"expectedStatus":"ok"}],"expectedStatus":"ok","minCitations":1,"maxStaleCitations":0,"maxUnsupportedClaims":0}
JSONL

bun src/cli.ts ask-replay sqlite-demo \
  --fixture "$tmp/sqlite-demo/tests/ask.jsonl" \
  --json \
  --root "$tmp"

bun src/cli.ts profile sqlite-demo --root "$tmp"
bun src/cli.ts doctor sqlite-demo --root "$tmp"
```

Expected sqlite-demo result:

- benchmark passes,
- saved answer artifact has planner/tool/citation/synthesis/quality trace,
- `runs` human output is readable,
- saved-run replay passes without provider credentials,
- fixture replay passes,
- `profile` reports `answer mode ready`,
- `doctor` reports answer readiness as ok.

## 3. Enterprise AI Compile Smoke

Create a fresh standard-depth Enterprise AI almanac:

```bash
export ALMANAC_ROOT="$(mktemp -d -t almanac-v080-enterprise-ai-smoke)"

bun src/cli.ts new "Enterprise AI" \
  --slug enterprise-ai-smoke \
  --profile mixed \
  --depth standard \
  --scope "Enterprise AI adoption and operations for organizations: AI governance, risk management, evaluation, security, compliance, deployment patterns, observability, and operating-model decisions for production AI systems." \
  --source "https://www.nist.gov/itl/ai-risk-management-framework" \
  --source "https://owasp.org/www-project-top-10-for-large-language-model-applications/" \
  --source "https://atlas.mitre.org/" \
  --root "$ALMANAC_ROOT"

bun src/cli.ts inspect enterprise-ai-smoke --root "$ALMANAC_ROOT"
bun src/cli.ts profile enterprise-ai-smoke --root "$ALMANAC_ROOT"
bun src/cli.ts benchmark enterprise-ai-smoke --root "$ALMANAC_ROOT"
```

If a provider timeout halts source evaluation, rerun from the failed stage:

```bash
bun src/cli.ts update enterprise-ai-smoke \
  --from-stage 02b-source-discovery-evaluator \
  --no-bump \
  --root "$ALMANAC_ROOT"
```

Expected Enterprise AI result:

- all compile stages complete,
- generated benchmark coverage is at or above 8 positive / 5 negative / 13
  total,
- benchmark passes with cited positives and negative refusals,
- `profile` and `doctor` expose answer readiness without hidden provider calls.

## 4. Real Provider Ask Smoke

Run only when `ANTHROPIC_API_KEY` is available:

```bash
test -n "$ANTHROPIC_API_KEY" || {
  echo "ANTHROPIC_API_KEY is required for real provider ask smoke"
  exit 1
}

bun src/cli.ts ask enterprise-ai-smoke \
  "What governance checks should precede production rollout?" \
  --save \
  --label rc-real-provider \
  --json \
  --root "$ALMANAC_ROOT"

bun src/cli.ts runs enterprise-ai-smoke \
  --kind answer \
  --latest \
  --json \
  --root "$ALMANAC_ROOT"

answer_id="$(
  bun src/cli.ts runs enterprise-ai-smoke \
    --kind answer \
    --latest \
    --json \
    --root "$ALMANAC_ROOT" \
    | jq -r '.runs[0].runId'
)"

bun src/cli.ts runs enterprise-ai-smoke \
  "$answer_id" \
  --json \
  --root "$ALMANAC_ROOT"

bun src/cli.ts ask-replay enterprise-ai-smoke \
  --from-runs \
  --label rc-real-provider \
  --json \
  --root "$ALMANAC_ROOT"

bun src/cli.ts doctor enterprise-ai-smoke --root "$ALMANAC_ROOT"
```

Expected real-provider result:

- answer status is either `ok` with citations or a correct abstention,
- saved answer artifact includes trace and quality sections,
- saved answer replay passes without another provider call,
- latest answer status and quality gate appear in `profile`/`doctor`.
